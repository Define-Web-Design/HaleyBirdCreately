### **Version 1: Comprehensive Prompt for Workflow Architect (Initial Iteration)**

> **Prompter to Use:**  
> Please use the **Workflow Architect** prompter. This role excels at designing and implementing complex UI components—such as modals, filtering tools, and keyword frameworks—integrated within cohesive workflows. While you may reference insights from the Creative Symbiosis Architect, Content Alchemist, and Platform Perfectionist when needed, begin with the Workflow Architect.
> 
> **Prompt:**
> 
> Haley, we are building a groundbreaking, fully integrated system that goes well beyond basic modals and filtering. Our goal is to provide a highly advanced UI that empowers users with customizable modals, intuitive tool panels, robust filtering options, and intelligent keyword management. This solution must be aesthetically stunning and technically robust, meeting current needs and anticipating future growth.
> 
> **Required functionalities:**
> 
> 1. **Modals & Pop-Up Tools**
>    - **Dynamic Modals:** Develop modals with multiple states (open, collapsed, expanded) using animation libraries (e.g., Framer Motion). Include customizable headers, footers, and content sections. Ensure responsiveness and full ADA compliance with hidden screen-reader texts.
>    
>    - **Embedded Tool Panels:** Create tool panels within modals that provide mini-forms, search fields, and real-time data updates, so users can adjust parameters in context.
>    
>    - **Code Example:**
>      ```tsx
>      import { motion } from 'framer-motion';
>      import React from 'react';
>      
>      const DynamicModal = ({ isOpen, onClose, header, content, footer }) => {
>        if (!isOpen) return null;
>        return (
>          <motion.div
>            initial={{ opacity: 0 }}
>            animate={{ opacity: 1 }}
>            exit={{ opacity: 0 }}
>            className="modal-overlay"
>            role="dialog"
>            aria-labelledby="modal-header"
>            aria-describedby="modal-content"
>            tabIndex={-1}
>          >
>            <div className="modal-container">
>              <header id="modal-header" className="modal-header">
>                {header}
>                <button onClick={onClose} aria-label="Close Modal">X</button>
>              </header>
>              <section id="modal-content" className="modal-content">
>                {content}
>              </section>
>              <footer className="modal-footer">
>                {footer}
>              </footer>
>            </div>
>          </motion.div>
>        );
>      };
>      export default DynamicModal;
>      ```
> 
> 2. **Advanced Filtering Options**
>    - **Multi-Faceted Filters:** Build filtering components (sliders, checkboxes, dropdowns) to let users refine content by keywords, moods, timestamps, etc. Provide immediate animated feedback.
>    
>    - **Code Example:**
>      ```tsx
>      import React, { useState } from 'react';
>      
>      const MultiFacetFilter = ({ onFilterChange }) => {
>        const [filters, setFilters] = useState({ keyword: '', mood: '' });
>        const updateFilters = (key, value) => {
>          const newFilters = { ...filters, [key]: value };
>          setFilters(newFilters);
>          onFilterChange(newFilters);
>        };
>        return (
>          <div className="filter-panel">
>            <input
>              type="text"
>              placeholder="Search by keywords..."
>              onChange={(e) => updateFilters('keyword', e.target.value)}
>            />
>            <select onChange={(e) => updateFilters('mood', e.target.value)}>
>              <option value="">All</option>
>              <option value="joyful">Joyful</option>
>              <option value="calm">Calm</option>
>              <option value="nostalgic">Nostalgic</option>
>            </select>
>          </div>
>        );
>      };
>      export default MultiFacetFilter;
>      ```
> 
> 3. **Keyword Frameworks**
>    - **Dynamic Keyword Generation:** Enable an AI-driven system that analyzes content metadata to suggest relevant keywords. Allow users to manage these keywords interactively.
>    
>    - **Interface Example:**
>      ```tsx
>      const KeywordPanel = ({ keywords, onUpdate }) => {
>        return (
>          <div className="keyword-panel">
>            <h4>Suggested Keywords</h4>
>            <ul>
>              {keywords.map((keyword, index) => (
>                <li key={index}>
>                  <span>{keyword.text}</span>
>                  <button onClick={() => onUpdate(keyword)}>Edit</button>
>                </li>
>              ))}
>            </ul>
>          </div>
>        );
>      };
>      export default KeywordPanel;
>      ```
> 
> 4. **Integration and Aesthetic Consistency**
>    - Ensure all components use a unified design language, adaptive theming (via CSS variables), and support for hidden accessible texts (using `.visually-hidden` CSS classes).
> 
> 5. **Testing and Future Scalability**
>    - Define unit, integration, and end-to-end tests using Jest and Cypress. Ensure ARIA standards are met and set a foundation for future expansion (like dashboards or premium features).
